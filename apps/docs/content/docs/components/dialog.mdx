## 개요

Seum의 Dialog는 세 가지 방식으로 사용할 수 있습니다.

- **`dialog.open()`** — 커스텀 모달을 여는 low-level API
- **`dialog.confirm()` / `dialog.alert()`** — 자주 쓰는 패턴을 위한 preset API
- **Dialog 컴포넌트** — 콘텐츠를 직접 정의할 때 쓰는 빌딩 블록

## CSS import

```ts
import 'seum/css/tokens'
import 'seum/css/dialog'
```

## 기본 사용법

### 콘텐츠 정의

```tsx
import { Dialog } from 'seum/dialog'

function MyModal({ onClose }) {
  return (
    <Dialog.Overlay>
      <Dialog.Content>
        <Dialog.Title>제목</Dialog.Title>
        여기서 내부 UI는 자유롭게 설계하세요
        <Dialog.Close asChild>
          <button onClick={onClose}>닫기</button>
        </Dialog.Close>
      </Dialog.Content>
    </Dialog.Overlay>
  )
}
```

### 어디서든 열기

```tsx
import { useDialog } from 'seum/dialog'

function Page() {
  const dialog = useDialog()

  return (
    <button onClick={() =>
      dialog.open(({ close }) => <MyModal onClose={close} />)
    }>
      열기
    </button>
  )
}
```

## 패턴 예시

### Modal

```tsx
function Modal({ children, onClose }) {
  return (
    <Dialog.Overlay>
      <Dialog.Content>
        {children}
        <Dialog.Close asChild>
          <button onClick={onClose}>닫기</button>
        </Dialog.Close>
      </Dialog.Content>
    </Dialog.Overlay>
  )
}

dialog.open(({ close }) => (
  <Modal onClose={close}>
    <h2>제목</h2>
    <p>내용</p>
  </Modal>
))
```

### Confirm preset

```tsx
const result = await dialog.confirm({
  title: '정말 삭제하시겠습니까?',
  description: '이 작업은 되돌릴 수 없습니다.',
  confirmLabel: '삭제',
  cancelLabel: '취소',
  tone: 'danger',
  loading: {
    title: '삭제 중입니다',
    confirmLabel: '처리 중',
  },
  success: {
    title: '삭제가 완료되었습니다',
    confirmLabel: '확인',
  },
  error: {
    title: '삭제하지 못했습니다',
    confirmLabel: '다시 시도',
    cancelLabel: '닫기',
  },
  onConfirm: () => deletePost(),
})

if (result.status !== 'confirmed') return
```

`confirm()`의 결과는 세 가지로 나뉩니다.

- `confirmed` — 확인 버튼을 통해 명시적으로 완료됨
- `cancelled` — confirm preset의 취소 버튼을 눌러 의도를 명시함
- `dismissed` — ESC, overlay click, 닫기 버튼처럼 값을 선택하지 않고 닫힘

## useDialog options

```tsx
dialog.open(render, {
  overlay: true,            // 오버레이 렌더링 여부 (기본: true)
  modal: true,              // 배경 인터랙션 차단 (기본: true)
  scrollLock: true,         // body 스크롤 잠금 (기본: true)
  closeOnOverlayClick: true // 오버레이 클릭 시 닫기 (기본: true)
})
```

`overlay: false`이면 `Dialog.Overlay`는 backdrop DOM을 만들지 않고 children만 그대로 렌더링합니다.

`dialog.open()`은 항상 handle을 반환합니다.

- `handle.result`로 결과를 기다릴 수 있습니다.
- `handle.update()`로 같은 모달 인스턴스의 데이터를 바꿀 수 있습니다.
- `handle.close()`로 명시적으로 닫을 수 있습니다.

단순 확인/알림은 `dialog.confirm()` / `dialog.alert()`를 쓰는 편이 더 자연스럽습니다.

- `confirm()`은 내부적으로 `confirm | loading | success | error` 단계를 관리합니다.
- `alert()`는 단일 확인 버튼이 필요한 공지성 모달에 맞습니다.
- `flow()`는 이런 preset을 구현하거나 확장할 때만 쓰는 advanced API로 생각하면 됩니다.

## 스타일링

`data-state` 속성으로 열림/닫힘 상태를 CSS에서 처리합니다.
닫힐 때는 `open -> closed`로 먼저 전환되고, exit transition 또는 animation이 끝난 뒤 unmount됩니다.

```css
[data-seum-dialog-overlay] {
  position: fixed;
  inset: 0;
  background: rgba(0, 0, 0, 0.5);
}

[data-seum-dialog-content] {
  position: fixed;
  top: 50%;
  left: 50%;
  transform: translate(-50%, -50%);
  background: white;
  border-radius: 8px;
  padding: 24px;
}

[data-seum-dialog-content][data-state='open'] {
  animation: dialogIn var(--seum-duration-normal) ease;
}

[data-seum-dialog-overlay][data-state='closed'] {
  animation: overlayOut 140ms ease-in;
}

[data-seum-dialog-content][data-state='closed'] {
  animation: dialogOut 140ms ease-in;
}
```

## 접근성

자동으로 처리되는 사항:

- `role="dialog"`, `aria-modal="true"` 부여
- 열리면 Content 내 첫 포커스 가능 요소로 이동
- 닫히면 열기 버튼으로 포커스 복귀
- `modal=true`일 때 포커스 트랩
- `Dialog.Title` 사용 시 `aria-labelledby` 자동 연결

## 키보드

| 키 | 동작 |
|----|------|
| `Escape` | 닫기 |
| `Tab` | Content 내부에서 포커스 순환 |
| `Shift+Tab` | 역방향 포커스 순환 |
