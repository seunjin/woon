## 설치

```bash
npm install seum
# or
pnpm add seum
# or
yarn add seum
```

## CSS 설정

Seum은 기능적 CSS(position, z-index 등)만 제공합니다. 앱 진입점에서 import 합니다.

```ts
// 전체 import
import 'seum/css'

// 또는 필요한 것만
import 'seum/css/tokens'
import 'seum/css/dialog'
```

## Provider 설정

`useDialog` 등 imperative API를 사용하려면 앱 루트에 `SeumProvider`를 추가합니다.

```tsx
import { SeumProvider } from 'seum'
import 'seum/css'

function App() {
  return (
    <SeumProvider>
      <YourApp />
    </SeumProvider>
  )
}
```

## 첫 Dialog 열기

```tsx
import { useDialog, Dialog } from 'seum/dialog'

function MyModal({ onClose }) {
  return (
    <Dialog.Overlay>
      <Dialog.Content>
        <p>안녕하세요!</p>
        <Dialog.Close asChild>
          <button onClick={onClose}>닫기</button>
        </Dialog.Close>
      </Dialog.Content>
    </Dialog.Overlay>
  )
}

function Page() {
  const dialog = useDialog()

  return (
    <button onClick={() =>
      dialog.open(({ close }) => <MyModal onClose={close} />)
    }>
      열기
    </button>
  )
}
```

`dialog.open()`은 handle을 반환합니다. 커스텀 모달은 `open()`으로 열고, 일반적인 확인/알림은 `dialog.confirm()` / `dialog.alert()`를 쓰는 편이 더 자연스럽습니다.

```tsx
const result = await dialog.confirm({
  title: '문서를 삭제할까요?',
  description: '이 작업은 되돌릴 수 없습니다.',
  confirmLabel: '삭제',
  cancelLabel: '취소',
  tone: 'danger',
  onConfirm: () => deletePost(),
})

if (result.status !== 'confirmed') return
```

닫힐 때는 즉시 unmount되지 않고 `data-state="closed"`로 한 번 전환됩니다.
그래서 `closed` 상태에 exit animation을 걸면 애니메이션이 끝난 뒤 DOM에서 제거됩니다.
