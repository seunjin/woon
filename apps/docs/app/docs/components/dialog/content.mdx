## 사용법

### 개요

```ts
import { useDialog } from '@woon/core'
import { Dialog, dialogPlugin } from '@woon/core/dialog'
```

Dialog는 두 가지 방식으로 사용할 수 있습니다:

- **명령형 API** — `useDialog()` 훅으로 다이얼로그를 열고 결과를 받습니다
- **프리셋 함수** — [`alert()`](/docs/components/dialog/alert), [`confirm()`](/docs/components/dialog/confirm)으로 간단한 알림/확인을 즉시 표시합니다

<Callout type="note">
  `useDialog()`는 WoonProvider가 필요합니다. `dialogPlugin()`을 Provider에 추가해야 합니다.
</Callout>

```tsx
import { WoonProvider } from '@woon/core'
import { dialogPlugin } from '@woon/core/dialog'

export function Providers({ children }: { children: React.ReactNode }) {
  return (
    <WoonProvider plugins={[dialogPlugin()]}>
      {children}
    </WoonProvider>
  )
}
```

### 기본 사용법

`useDialog()` 훅에서 `dialog.open()`을 호출하면 다이얼로그가 열립니다.
render 함수에 `close`, `resolve`, `update` 등의 컨텍스트가 전달됩니다.

```tsx
import { useDialog } from '@woon/core'
import { Dialog } from '@woon/core/dialog'

function MyPage() {
  const dialog = useDialog()

  return (
    <button
      onClick={() =>
        dialog.open(({ close }) => (
          <Dialog.Root>
            <Dialog.Overlay />
            <Dialog.Content>
              <Dialog.Title>제목</Dialog.Title>
              <Dialog.Description>설명 텍스트</Dialog.Description>
              <Dialog.Close>닫기</Dialog.Close>
            </Dialog.Content>
          </Dialog.Root>
        ))
      }
    >
      열기
    </button>
  )
}
```

### 값 반환

`dialog.open()`은 핸들 객체를 반환합니다. `result` Promise로 다이얼로그의 결과를 받을 수 있습니다.

```tsx
const handle = dialog.open<undefined, string>(({ resolve, close }) => (
  <Dialog.Root>
    <Dialog.Overlay />
    <Dialog.Content>
      <Dialog.Title>선택하세요</Dialog.Title>
      <button onClick={() => resolve('yes')}>확인</button>
      <button onClick={close}>취소</button>
    </Dialog.Content>
  </Dialog.Root>
))

const result = await handle.result
// result.status === 'resolved' → result.value === 'yes'
// result.status === 'dismissed' → 오버레이 클릭이나 ESC로 닫힘
```

### 중첩 다이얼로그

다이얼로그 안에서 `dialog.open()`을 다시 호출하면 스택으로 쌓입니다.
ESC 키는 가장 위의 다이얼로그만 닫습니다. `closeAll()`로 전체를 닫을 수 있습니다.

```tsx
const dialog = useDialog()

dialog.open(({ close }) => (
  <Dialog.Root>
    <Dialog.Overlay />
    <Dialog.Content>
      <button onClick={() => dialog.open(/* 두 번째 다이얼로그 */)}>
        다음 다이얼로그
      </button>
    </Dialog.Content>
  </Dialog.Root>
))

// 전체 닫기
dialog.closeAll()
```

### 프리셋

Dialog 시스템 위에서 동작하는 명령형 프리셋 함수입니다.
WoonProvider 없이 어디서든 호출할 수 있습니다.

- **[alert()](/docs/components/dialog/alert)** — 사용자 확인이 필요한 단순 알림
- **[confirm()](/docs/components/dialog/confirm)** — 확인/취소 선택 + 비동기 플로우

## API Reference

### 컴파운드 컴포넌트

아래 primitive를 조합해 Dialog 구조를 만듭니다.

| 컴포넌트 | 설명 | 기본 요소 |
|----------|------|-----------|
| `Dialog.Root` | 다이얼로그 루트 컨테이너. `options` prop으로 기본값 설정 | — |
| `Dialog.Overlay` | 반투명 배경. 클릭 시 닫기 (`closeOnOverlayClick`) | `<div>` |
| `Dialog.Content` | 다이얼로그 본문. 포커스 트랩, ARIA 속성 자동 적용 | `<div role="dialog">` |
| `Dialog.Title` | 제목. `aria-labelledby` 자동 연결 | `<h2>` |
| `Dialog.Description` | 설명. `aria-describedby` 자동 연결 | `<p>` |
| `Dialog.Close` | 닫기 버튼. 클릭 시 `close()` 호출 | `<button>` |

`Dialog.Title`, `Dialog.Description`, `Dialog.Close`는 `asChild` prop을 지원합니다.

```tsx
<Dialog.Close asChild>
  <button className="my-close-button">X</button>
</Dialog.Close>
```

### DialogOptions

<Callout type="tip">
  런타임 옵션은 `Dialog.Root options={{ ... }}` 또는 `dialog.open(render, options)`의 두 번째 인자로 전달합니다.
</Callout>

| 옵션 | 타입 | 기본값 | 설명 |
|------|------|--------|------|
| `overlay` | `boolean` | `true` | 반투명 배경 표시 |
| `trapFocus` | `boolean` | `true` | 포커스를 다이얼로그 내부에 가둠 |
| `scrollLock` | `boolean` | `true` | body 스크롤 잠금 |
| `closeOnOverlayClick` | `boolean` | `true` | 배경 클릭 시 닫기 |

**우선순위** (낮음 → 높음):

1. 라이브러리 기본값
2. `Dialog.Root options` prop
3. `dialog.open()` 명시 옵션

`overlay`, `trapFocus`, `scrollLock`을 모두 끄면 사이드 패널이나 드로어 패턴을 구현할 수 있습니다.

### 스타일

기본 스타일은 `@woon/core/css/dialog`로 적용됩니다. 컴포넌트 파일 상단 또는 전역 CSS에서 한 번만 import하세요.

```ts
import '@woon/core/css/dialog'
```

**자주 바꾸는 부분은 선택자 하나만 override하면 됩니다.**

배경, 라운드, 그림자:

```css
[data-woon-dialog-content] {
  background: #1a1a1a;
  color: #fff;
  border-radius: 12px;
  box-shadow: 0 20px 60px rgba(0, 0, 0, 0.4);
}
```

오버레이 색상 / 블러:

```css
[data-woon-dialog-overlay] {
  background: rgba(0, 0, 0, 0.6);
  backdrop-filter: blur(4px);
}
```

애니메이션 제거:

```css
[data-woon-dialog-overlay],
[data-woon-dialog-content] {
  animation: none !important;
}
```

### 접근성

Dialog는 **포커스 트래핑, ESC 닫기, 포커스 복원, body 스크롤 락**을 자동으로 처리합니다.

**키보드 인터랙션**

| 키 | 동작 |
|----|------|
| `Tab` | 다이얼로그 내부 포커스를 순환합니다 |
| `Shift + Tab` | 역방향으로 포커스를 순환합니다 |
| `Esc` | 가장 위에 떠 있는 다이얼로그만 닫습니다 |

**ARIA** — `role="dialog"`, `aria-modal="true"`, `aria-labelledby`(Title), `aria-describedby`(Description)가 컴파운드 컴포넌트에 따라 자동 연결됩니다.

### data 속성

| 속성 | 대상 | 값 |
|------|------|------|
| `data-woon-dialog-overlay` | Overlay | — |
| `data-woon-dialog-content` | Content | — |
| `data-state` | Overlay, Content | `"open"` 또는 `"closed"` |

상태 기반 스타일링 예시:

```css
[data-woon-dialog-content][data-state="open"] {
  animation: my-dialog-in 200ms ease-out;
}
```
