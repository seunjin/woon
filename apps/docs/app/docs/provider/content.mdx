```ts
import { WoonProvider } from '@woon/core'
import { dialogPlugin } from '@woon/core/dialog'
import { toastPlugin } from '@woon/core/toast'
```

## WoonProvider

`WoonProvider`는 overlay 기능의 렌더 위치와 전역 설정을 연결하는 컴포넌트입니다.
앱 진입점에 한 번만 등록하면 됩니다.

```tsx
export function Providers({ children }: { children: React.ReactNode }) {
  return (
    <WoonProvider plugins={[dialogPlugin(), toastPlugin()]}>
      {children}
    </WoonProvider>
  )
}
```

`plugins` prop에 사용할 플러그인 배열을 전달합니다. 순서는 상관없습니다.

<Callout type="note">
  `alert()`, `confirm()`, `toast()`는 `WoonProvider` 없이도 호출할 수 있습니다. 단, Provider가 없으면 화면에 렌더되지 않습니다.
</Callout>

## 플러그인

### dialogPlugin

`useDialog()`, `alert()`, `confirm()`을 사용하려면 등록해야 합니다.

```tsx
dialogPlugin({
  zIndex: 200,
  scrollTarget: '#main',
  render: {
    alert: MyAlert,
    confirm: MyConfirm,
  },
})
```

`scrollTarget`은 `body { height: 100dvh }` 같은 레이아웃에서 body가 스크롤을 담당하지 않을 때 실제 스크롤 컨테이너를 지정하는 용도입니다. `dialog.open(render, { scrollTarget })` 옵션으로 호출별 오버라이드도 가능합니다.

### toastPlugin

`toast()`를 사용하려면 등록해야 합니다.

```tsx
toastPlugin({
  position: 'bottom-right',
  maxVisible: 3,
  zIndex: 9000,
})
```
